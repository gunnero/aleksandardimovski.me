# Security Policy

## Supported version

Security updates are provided for the version currently running in production. Older revisions, development branches, and archived artifacts are not supported.

## Reporting a vulnerability

Report suspected vulnerabilities privately by email to [aleksandar.dimovski@me.com](mailto:aleksandar.dimovski@me.com). Do not open a public GitHub issue, discussion, or pull request for a security report.

Please include:

- a clear description of the issue and its potential impact;
- the affected page, route, component, or revision;
- reproducible steps or a minimal proof of concept;
- relevant browser, operating-system, or client details;
- suggested remediation, if known.

Do not include credentials, personal data, server topology, infrastructure details, environment files, logs containing secrets, or other sensitive production data. Redact sensitive evidence and wait for a secure exchange method if additional material is required.

Receipt will be acknowledged and the report will be assessed. Further updates will depend on the severity, reproducibility, and remediation work required; no fixed response or resolution time is promised.

Please keep the report and related findings confidential until a remediation or coordinated disclosure plan has been agreed.

Linked projects and repositories are governed by their own `SECURITY.md` files. Report an issue to the repository that contains the affected code.
