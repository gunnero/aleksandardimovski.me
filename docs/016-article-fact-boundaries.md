# Article Fact Boundaries

## Publicly supported BuildIQ evidence

BuildIQ is an active-development product using Python, FastAPI, Starlette, PostgreSQL, React, TypeScript, Vite, and Vitest. The reviewed baseline records 127 backend tests, 45 frontend tests, pip-audit, npm audit, Gitleaks, CI, route-level lazy loading, and a primary-bundle reduction from approximately 520 kB to 367.94 kB.

Do not describe BuildIQ as Laravel-based, claim years of Python experience, invent a Python employment role, or claim customers, revenue, adoption, or commercial success.

## Legacy case-study boundary

Hera Backoffice and Nema30 Backoffice are distinct sanitized examples. Public writing may discuss workflow discovery, Git adoption, backups, compatibility, security, controlled deployment, and rollback. It must not combine the applications into a fictional workflow or disclose private users, data, internal URLs, databases, infrastructure, source code, or client-sensitive rules.

## Deployment boundary

Public writing may describe exact-commit releases, runtime checks, protected environment configuration, `.well-known` preservation, dependency installation, asset builds, migrations, caches, permissions, service health, TLS, logs, clean Git state, and rollback.

Do not publish server hostnames, IP addresses, usernames, absolute production paths, certificate topology, backup locations, SSH commands, or private domains.

## Personal boundary

Do not add LinkedIn, date of birth, nationality, private contact information, or unverified chronology. Existing public email, telephone, location, GitHub profile, titles, and reviewed employment chronology remain governed by the resume source of truth.
