# Resume QR code policy

QR codes are permitted only in the downloadable and printable resume, never as substitutes for normal website links. Every code must have adjacent project text and a selectable, clickable URL so no information is QR-only.

Codes are generated locally from verified HTTPS destinations, embedded as deterministic high-resolution assets, rendered dark on white with a quiet zone, and checked with two independent decoders. Tracking services, URL shorteners, dynamic redirects, remote image dependencies, and private repository destinations are prohibited.

BuildIQ points to `https://aleksandardimovski.me/projects/buildiq`; MediaHub points to `https://aleksandardimovski.me/projects/mediahub`. The portfolio and Kalveri use clickable text because adding more codes would reduce scanability. The governing acceptance test is still an exactly two-page, selectable, ATS-friendly A4 PDF with a white grayscale-safe print background.
