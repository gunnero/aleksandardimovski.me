# Public repository link policy

A repository link may be published only when the repository is public, represents the named project, loads its default branch, provides a README, and has passed a confidentiality and public-evidence review. Link labels identify the project and destination; new-tab links use `noopener noreferrer` and an accessible external-link indication.

Approved links are limited to `gunnero/aleksandardimovski.me`, `gunnero/BuildIQ`, `gunnero/mediahub`, and `gunnero/kalveri`. Public status is not sufficient by itself: quarantined, empty, client, private, or otherwise unapproved repositories remain unlinked until a separate remediation program clears every promotion gate.

Private work uses the label “Private commercial repository” and offers a sanitized public case study. The interface never renders disabled repository controls, private URLs, stars, activity counters, or unsupported maturity claims.
